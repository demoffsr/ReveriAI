# ReveriAI — AI Services

Reference for Supabase Edge Functions, OpenAI integration, and AI pipelines.
Load this file when working on AI features, edge functions, or Supabase integration.

**Deploy edge functions:**
`~/bin/supabase functions deploy <name> --project-ref bvydopjjndfgbhjczyis`

**Supabase project ref:** `bvydopjjndfgbhjczyis`

---

- **AI title generation:** After saving a dream, `DreamAIService.generateTitleInBackground()` calls a Supabase Edge Function (`generate-dream-title`) which uses OpenAI GPT-4o-mini to generate a 3-5 word concrete title (e.g. "Чай с говорящей кошкой", "Endless Library of Lives"). Runs in background `Task { @MainActor in }`, uses `PersistentIdentifier` + `ModelContainer` for safe SwiftData access. Title appears in journal after ~1-2s. If AI unavailable — title stays empty, no crashes. Locale from `SpeechLocale` picker determines prompt language (Russian vs English). `SupabaseService` holds shared `SupabaseClient` (15s timeout). **Config:** `SupabaseConfig` reads from `Bundle.main.infoDictionary` (keys injected from `Secrets.xcconfig` via `Info.plist`). Edge Function at `supabase/functions/generate-dream-title/index.ts`.
- **AI image generation:** Two-stage pipeline for high-quality dream visualization. (1) **Questions:** `DreamAIService.generateQuestions()` → Edge Function `generate-dream-questions` → GPT-4o-mini generates 3 personalized visual detail questions based on dream text (landscape, lighting, atmosphere, emotions, colors). (2) **Art prompt:** User answers → GPT-4o-mini writes detailed 150-200 word English art prompt (composition, lighting, palette, atmosphere, surreal elements, cinematic style). (3) **Image gen:** Art prompt → `gpt-image-1` model (1024×1024, `quality: high`, `moderation: low`) → returns b64_json → uploaded to Supabase Storage bucket `dream-images` → public URL saved to `Dream.imageURL`. Edge Function `generate-dream-image` handles full pipeline. Questions flow: native `.sheet` with 3 TextFields, Generate/Skip buttons, 0.3s dismiss delay before generation to avoid race condition. Thumbnail: 74×74 in DreamDetailView header, shimmer during generation, AsyncImage with fullscreen button overlay when loaded.
- **AI dream interpretation (v2 — multi-framework):** Scientific multi-framework analysis via GPT-4o. `DreamAIService.generateInterpretation()` → Edge Function `generate-dream-interpretation` → OpenAI GPT-4o (max_tokens: 2500, temp: 0.65, `response_format: json_object`). **6 psychological frameworks** (model picks 3 most relevant): Hartmann (emotional metaphor, always first), Jung (archetypes), Fromm (symbol categories), Revonsuo (threat simulation), Barrett (problem solving), Gestalt (character projections). Input: dream text + locale + emotions array. Output: structured JSON (`InterpretationData`): `emotionalCore` (title + body), `frameworks[3]` (id, title, SF Symbol icon, body), `symbols[]` (symbol + meaning), `reflection[3]` (Hill-model questions), `synthesis`. JSON stored as string in `Dream.interpretation` (field unchanged). **Client model:** `InterpretationData.swift` — Codable struct with `isV2` computed property, `parse(from:)` static method. **Backward compatibility:** Old plain-text interpretations render via legacy regex parser (`parseInterpretation()`, `parseAndStyleSections()`). v1 JSON fallback wraps raw text. **UI (DreamDetailView):** Hero emotional-core card (accent border), 3 collapsible framework cards (SF Symbol + chevron, tap to expand/collapse with spring animation), symbols list (accent bullet), reflection card (numbered italic questions), synthesis paragraph. **Cost:** ~$0.014/interpretation (was $0.0006 with gpt-4o-mini). Deploy: `~/bin/supabase functions deploy generate-dream-interpretation --project-ref bvydopjjndfgbhjczyis`.
