# ReveriAI — Component Patterns

Reference for UI components, state machines, and architectural patterns.
Load this file when working on any UI component, recording flow, or navigation.

---

- **Cloud system:** 3 cloud Shape layers (Back/Mid/Front) normalized from SVG viewBox 390×159 using 0..1 coords. `CloudSeparator` composes them. `RecordView` controls sizing: `cloudHeight = 89`, `cloudOverhang = cloudHeight * 0.5`. **Closing clouds:** `CloudClosedShape` (inverted — flat top, wavy bottom, from `clouds_closed_new.svg` 390×140) slides down from above when `isTextFocused` to cover the header. VStack: solid `theme.cloudFront` fill + `CloudClosedShape` at `cloudHeight`. Lives inside `cloudLayer` overlay (above `CloudSeparator` z-order). Offset animation: `-(baseHeaderHeight + cloudOverhang)` → `0`. **Animation split:** Cloud/content movement uses `.spring(duration: 0.45, bounce: 0.0)` via `.animation(_:value: isTextFocused)` at body call sites. Header gradient + title + CloudSeparator opacity use separate `@State headerContentVisible` driven by `onChange(of: isTextFocused)` with `withAnimation(.easeOut(duration: 0.1))` — avoids conflicting `.animation` modifiers on the same view (outer `.animation` overrides inner for `.frame`). CloudSeparator fades with `headerContentVisible` (fast 0.1s) to prevent darker back/mid cloud layers showing through CloudClosedShape valleys. **Gradient frame:** `headerHeight + cloudOverhang - 8` — stops 8pt before cloud bottom to prevent photo bleeding through cloud shape valleys.
- **CelestialIcon:** 2 glow rings (102pt/84pt) + main circle (65pt) with gradient stroke + warm shadow. Day = custom SVG sun (`SunIcon` asset), Night = SF Symbol `moon.fill`.
- **Header background:** `DreamHeader` uses `Image("BackgroundDaylight")` with `.resizable().aspectRatio(contentMode: .fill)`, overlaid by a black-to-transparent `LinearGradient` (Figma devmode values) + `StarsCanvas`. In `RecordView`, the header frame is `headerHeight + cloudOverhang - 8` with `.clipped()` so the photo extends behind clouds but stops before the bumpy bottom edge (prevents bleed through cloud valleys).
- **Header layout:** Layered ZStack in RecordView — background, content, header image, title+icon, clouds+pill. Title shifts up on keyboard focus.
- **Mode switch pill:** `.glassEffect(.clear.interactive(), in: .capsule)` with white stroke overlay. Shows the *opposite* mode (i.e. what you can switch to). Custom SVG icons: `VoiceModeIcon`, `TextModeIcon`. Positioned at `bottomTrailing` of cloud layer with `offset(y: cloudOverhang + 30)`.
- **Save Dream button:** Appears in text mode when `canSave` is true, and in review mode at `bottomTrailing`. Glass effect + white stroke (same style as mode pill). Background: `theme.accent.opacity(0.1)`, text: `theme.accent` (100%). Icon: `CheckmarkIconAction` (original rendering, orange SVG).
- **Text input:** `TextModeView` uses `.tint(theme.accent)` for orange cursor/selection. Dynamic top padding: `16pt` when focused (keyboard up, text closer to clouds) / `36pt` when unfocused (room for Save Dream pill).
- **Tab bar:** Glass effect (`.reveriGlass(.capsule, interactive: false)` — same style as Generate Image/Interpret Dream buttons) + white stroke overlay. Custom SVG icons with `.renderingMode(.template)` + dynamic coloring: selected tab uses `theme.accent` (orange day/blue night), inactive uses `black.opacity(0.3)`. `AppTab` has `activeIcon`/`inactiveIcon` properties. Selected tab has `accentColor.opacity(0.15)` capsule background. Label: SF Pro 15 medium, accentColor. Animation: expand on tap → collapse after 1.5s via cancellable `Task`. Four-state body: normal tabs / recording controls / review controls / **detail dream controls** (separate white glass button when `isInDetailDreamTab`, no shared glass container — standalone card).
- **Tab switching (Record ↔ Journal):** `RootView` uses `ZStack` with `zIndex` only — NO `.opacity()` transition, NO `.animation()` on the ZStack. Обе вьюхи всегда `opacity 1`; `zIndex` определяет, кто сверху. Ghosting невозможен, т.к. обе вьюхи имеют полностью непрозрачный фон на весь экран (RecordView: `theme.cloudFront` + `.ignoresSafeArea()`, JournalView: `Color(.systemGroupedBackground)` + `.ignoresSafeArea(edges: .top)`). Визуальный feedback даёт таб-бар (spring-анимация label + иконки). `.allowsHitTesting(selectedTab == .xxx)` блокирует взаимодействие с неактивной вьюхой. **ВАЖНО:** не добавлять обратно `.opacity()` или `.animation(value: selectedTab)` — это вызовет ghosting.
- **Recording state machine:** `idle → recording → (transcript? text mode : reviewing) → idle`. States `isRecording`/`isPaused`/`isReviewing` live in `RootView`, flow via `@Binding` to `RecordView` and as values + closures to `ReveriTabBar`. `AudioRecorder` and `SpeechRecognitionService` also live in `RootView` (shared between RecordView and tab bar). Recording < 1s auto-discards.
- **Voice → text flow:** After recording stops, if transcript exists → copy to `viewModel.dreamText`, switch to text mode, delete audio file. User edits and saves as text dream. If no transcript → fall back to audio review mode.
- **Recording mode:** Tab bar: Stop icon always left, Pause/Play icon always right. Labels migrate: "Stop" label on left when recording, "Resume" label on right when paused. Transition via `.blurReplace` + `.spring`. 3D flip on Pause/Play icon. Mode pill hidden, timer text at `bottomLeading`. Waveform in content area.
- **Review mode (audio only):** Tab bar has two states: **idle** shows Play button (accent, labeled "Play") + Delete button (red trash icon, triggers confirmation alert "Delete recording?"). **Playback active** (`isPlayingPreview`) shows [SkipBack5Icon] — [Large Pause/Play in 56pt cream circle (`theme.accent.opacity(0.15)`)] — [SkipForward5Icon], with `.blurReplace` transition between states. `AudioRecorder` has `skipForward(seconds:)` / `skipBackward(seconds:)` methods. Cloud layer: review timer at `bottomLeading` (`00:00:00 — 00:05:23`), Save Dream button at `bottomTrailing`. Waveform shows playback animation when Play is pressed — scrolls through recorded bars at 60fps, synced with audio duration. Editable `TextEditor` with transcript from speech recognition (placeholder "Add dream description..." when empty). Saved as audio dream with transcript.
- **Audio recording:** `AudioRecorder` service (`@Observable`, NSObject, AVAudioPlayerDelegate). `AVAudioEngine` with input tap for recording (AAC 44.1kHz, `Documents/recordings/`). AVAudioPlayer for playback preview. Metering via `vDSP_maxmgv` + `vDSP_rmsqv` (Accelerate framework), peak/RMS blend (70/30), `cbrtf` curve. Raw curved level dispatched to MainActor — `WaveformBuffer` handles its own smoothing. `NSMicrophoneUsageDescription` in project build settings.
- **Speech recognition:** `SpeechRecognitionService` (`@Observable`) with 3-tier fallback: **Tier 1** `SpeechAnalyzer`/`SpeechTranscriber` (iOS 26+, on-device, no time limit, 37+ locales incl. ru_RU) → **Tier 2** `SFSpeechRecognizer` (fallback) → **Tier 3** WhisperKit (stub). Auto-selects best engine per locale. Exposes `transcribedText`, `partialText`, `stableText`, `latestText`, `isTranscribing`, `currentEngine` enum. `BufferConverter` handles audio format conversion for SpeechAnalyzer. `startTranscription(locale:audioStream:)` takes explicit `Locale` from user's picker selection. Both engines use `AudioBufferRelay` (NSLock-based pub/sub fan-out) for session restart loops — when a session ends (speech pause or SF 1-min limit), partial text is committed via `pauseTranscription()` and a new session subscribes to the relay. **SF silent reset detection:** SFSpeechRecognizer silently resets `bestTranscription.formattedString` on new utterance without sending `isFinal`. Detected by comparing first characters (case-insensitive) of consecutive results + text shrink heuristic. On detection, previous text is auto-committed. `lowercaseFirst` applied when appending segments — but skipped after sentence-ending punctuation (`.?!`) to preserve capitalization. **Simulator note:** SpeechAnalyzer is unavailable on iOS Simulator — falls back to SFSpeechRecognizer (Tier 2).
- **Punctuation post-processing:** Apple's `addsPunctuation` works for English but NOT for Russian. `punctuateSegment()` adds `.` or `?` at segment commit boundaries (speech pauses, session restarts, isFinal). Question detection: skips Russian filler words at the start (а, ну, и, но, да, так, вот, ведь, же), then checks if the first significant word is a question word (почему, где, кто, что, когда, сколько, etc.) → `?`. Also detects «ли» particle anywhere → `?`. No-op if text already has punctuation (English). Applied in `pauseTranscription()`, SF auto-commit, SF isFinal, and SA isFinal. SF also has `request.addsPunctuation = true` + `requiresOnDeviceRecognition` for English. **Performance:** `fillers` and `questionWords` are `private static let` sets — created once, reused on every call (hot path optimization).
- **Locale picker:** `SpeechLocale` enum (13 locales: RU, EN, DE, FR, ES, IT, PT, JA, KO, ZH, AR, TR, HI). Persisted via `@AppStorage("speechRecognitionLocale")`. Picker UI in `JournalHeader` — avatar circle shows 2-letter locale code, tapping opens `Menu` with all locales + checkmark. `RecordView` reads same `@AppStorage` key to pass locale to speech service.
- **Live captions:** SF Pro 15pt, tracking -0.23, lineSpacing 5. Latest in-progress word styled with `LinearGradient` (black → accent). Uses `Text` string interpolation with embedded styled `Text` views (NOT `Text + Text` which is deprecated in iOS 26). Wrapped in `ScrollView`, positioned directly under waveform with 8pt spacing. 100pt bottom padding for tab bar clearance.
- **Audio waveform:** `AudioWaveformView` uses `Canvas` + `TimelineView(.animation)` for 60fps rendering. Three modes: **recording** (bars grow left-to-right from live audio level), **review static** (shows playback position from `playbackProgress`), **playback** (smooth 60fps scroll driven by `TimelineView`, speed = `totalRecordedOffset / playbackDuration`). `WaveformBuffer` (plain class, NOT @Observable) stores bars — mutations don't trigger SwiftUI state diffs, no trim limit (all bars preserved for playback). New bars only generated during recording (`isAnimating`). Bar width 2pt, spacing 3.6pt, height 4–100pt (frame 120pt). Level smoothing: fast attack (0.3/0.7), moderate decay (0.8/0.2). `totalRecordedOffset` captured when recording ends via `onChange(of: isAnimating)`. Playback start/pause managed by `onChange(of: isPlayingBack)` — captures/restores scroll position. **Playback window:** During playback/review, playhead positioned at 20% from left edge (`visibleWidth = size.width * 0.2`) so scrolling is visible from the first second. During recording, full width used. **Skip sync:** `onChange(of: playbackProgress)` detects jumps (delta > 2%) and re-syncs `playbackAnimStartOffset`/`playbackAnimStartTime`. **Critical:** `animationStartTime` must be `TimeInterval?` (not `= 0`) — TimelineView fires before `onAppear`, causing `elapsed` overflow if initialized to epoch. `LiveWaveformView` wrapper in RecordView isolates observation of `audioRecorder.currentLevel` (recording) and `audioRecorder.playbackCurrentTime` (review) so only the small wrapper re-evaluates, not RecordView. **ViewBuilder pitfall:** `if/else` with `let` assignment inside `TimelineView` closure breaks generic inference — extract into a regular `func` instead.
- **EmotionTagBadge:** Shared inline capsule badge (`Components/EmotionPicker/EmotionTagBadge.swift`). HStack: emotion icon + `displayName` text, capsule background `emotion.color.opacity(0.15)`. Configurable `iconSize` (default 16) and `fontSize` (default 12). Used in DreamCard (16/12) and DreamDetailView (18/13). **NOT** the same as `EmotionBadge` (circle+emoji for picker).
- **DreamCard:** White card with 24pt corner radius, 14pt padding, spacing 14pt between cards. Layout: title (16pt semibold) + options menu (glass circle 32×32 with ellipsis icon) → `EmotionTagBadge` badges with -6pt top padding from title → audio waveform player OR text preview (13pt, 2 lines) → divider (0.5pt) → date row (CalendarSmallIcon 16×16 + date string 13pt). Options menu: Rename, Add to Folder (opens `FolderPickerSheet` via `.sheet`), Share, Delete (destructive). Entire card tappable via `onTap` callback → opens `DreamDetailView` in NavigationStack. `audioURL` computed from `dream.audioFilePath` via `Documents/recordings/{filename}` using `appendingPathComponent()`.
- **DreamDetailView:** Full-screen detail for a dream. Custom nav bar: back button (glass circle 44×44, `chevron.left`) + center "Dream" title (optional `folderName` subtitle) + options button (glass circle 44×44, `ellipsis`). No `GlassEffectContainer` — buttons independent. Fixed header: HStack with title (20pt semibold) + `EmotionTagBadge` badges on left, 74×74 dream image thumbnail on right (rounded 12pt, shows shimmer when generating, `CachedAsyncImage` when loaded, fullscreen button overlay). Below: date (CalendarSmallIcon 18×18 + `dreamFormatted`, `.black.opacity(0.35)`) + segmented `Picker` (Dream/Meaning). **Dream tab:** scrollable text (15pt, lineSpacing 4, tracking -0.23, `.black.opacity(0.8)`). **Meaning tab:** 5 states — (1) short/empty text: centered "Add a text description..." (2) placeholder: centered EmotionJoyful 60×60 + "Curious what it means?" + "Discover the symbols\nand emotions hidden within" (3) generating: ProgressView + "Interpreting dream..." (4) error: triangle + message + "Try again" button (5) interpretation: scrollable parsed sections with titles (16pt medium) + body text (subheadline). Interpretation sections cached in `@State`, parsed once in `onAppear` and `onChange(of: detailState.hasInterpretation)` — regex parsing NOT run on every body render. Markdown parsing: `**bold**` → semibold, bullet points (`•`/`-`) with indent, numbered headers split into sections. Empty states (1-4) centered vertically between segment control and tab bar button, rendered outside ScrollView. Background `systemGroupedBackground`. `.toolbar(.hidden, for: .navigationBar)`. Bottom padding 100pt for tab bar. **Tab bar button** controlled by `DetailDreamState.tabBarMode` enum: `.generateImage`/`.generateImageAgain` (Dream tab) shows "Generate Image"/"Generate Again" + GenerateImageIcon, `.interpretDream` (Meaning tab, no interpretation yet) shows "Interpret Dream" + InterpretIcon, `.none` (interpretation exists or text too short) hides button. Double-tap protection via `isGeneratingInterpretation` guard. Questions sheet flow (image gen): tap Generate → GPT-4o-mini generates 3 personalized visual questions → native `.sheet` with text fields → answers enrich DALL-E prompt. Interpretation trigger via `detailState.interpretTrigger` boolean toggle. `folderName: String? = nil` param — folders to be added later, model not modified.
- **Folders system:** `DreamFolder` (`@Model`) with `id`, `name`, `createdAt`, `@Relationship(inverse: \Dream.folder) var dreams: [Dream]`. `Dream` has `var folder: DreamFolder?` (one-to-many). Registered in `ModelContainer(for: [Dream.self, DreamFolder.self])`. **FolderCard:** white card 20pt radius, border 1px `black.opacity(0.1)`, padding 14pt, minHeight 117pt. Top area: empty folder shows "Add dreams" italic 13pt centered in 32pt frame; non-empty shows top 3 most frequent emotions as overlapping 32×32 circles (18pt offset, `.ultraThinMaterial` background). Emotion icons cached in `@State` via `.task(id: folder.dreams.count)` to avoid recomputing on every render. Bottom: name (16pt medium) + "N Dreams" (13pt) + ellipsis menu button (30×30 glass circle, `HStack(alignment: .top)`). Menu: Rename, Delete. Grid: 2-column `LazyVGrid`, spacing 14pt. **FolderDetailView:** custom nav bar (back 44×44 glass circle + center title + AddIcon 28×28 glass circle in `GlassEffectContainer`), `FolderSearchBar` under nav bar, dream cards list. `.toolbar(.hidden, for: .navigationBar)`. Add button opens `AddDreamsToFolderSheet`. Filtered dreams cached in `@State`, updated via `.onChange(of: searchText)` and `.onChange(of: folder.dreams.count)`. **AddDreamsToFolderSheet:** all dreams list with search, each row shows title + date + toggle checkmark. Added dreams: `CheckmarkIconBig` tinted `theme.accent` + accent tint background/border. Not added: `CheckmarkEmptyIcon` tinted `black.opacity(0.15)`. Tap toggles `dream.folder`. **FolderPickerSheet:** shown from DreamCard "Add to Folder" menu. Grid of FolderCards, search bar, FolderCreateIcon button in toolbar. **FolderSearchBar:** shared component — white background capsule, dark magnifyingglass icon, 44pt height, 1px border `black.opacity(0.1)`. **New Folder alert:** native `.alert` with TextField, triggered from JournalHeader "New Folder" button via `showNewFolderAlert` binding. State lives in `JournalView`.
- **Recording Live Activity (lock screen):** Shows live waveform + timer + stop button during voice recording. **Architecture:** `RecordingActivityAttributes` (ActivityAttributes, `ContentState` with `isPaused`, `recordingStartDate`, `pausedElapsedSeconds`, `levels: [Float]`, `levelStartIndex: Int`). `LiveActivityManager` (@Observable, lives in RootView) handles lifecycle (`startRecording()`/`pause()`/`resume()`/`end()`) and level sampling. **Level sampling pipeline:** `startLevelSampling()` polls `audioRecorder.currentLevel` every 250ms, applies smoothing (fast attack 0.3/0.7, slow decay 0.8/0.2), appends to `levelBuffer` (max 40 bars), pushes via `Activity.update()`. **Lock screen banner:** HStack — `LiveWaveformWidget` (left) + timer + glass stop button (right). Background: LinearGradient (`palette.gradientStart`→`gradientEnd`) + NoiseTexture tile, 20pt rounded rect with 0.5pt white stroke. Stop button links to `reveri://stop-recording`. **`LiveWaveformWidget`:** 27 fixed positions, single `ForEach(0..<27, id: \.self)`. Each position renders one `Capsule` that varies height/color: positions with data show `palette.accent` at computed height, unfilled positions show `Color.white.opacity(0.25)` at `minHeight` (6pt). Bars fill **left-to-right** (oldest data at position 0, placeholders on the right). After all 27 filled, the data window slides — heights animate smoothly at each position. Left-edge gradient mask (30pt fade) softens oldest bars. Pause: 40% opacity. Bar width 4pt, spacing 3.628pt, height range 6–62pt, frame 209×62pt. `levelStartIndex` in model is vestigial (no longer used for bar IDs). **Dynamic Island:** Compact: waveform SF Symbol + timer. Expanded: waveform icon (leading) + timer (center) + stop button (trailing).
- **Card waveform player:** `DreamCardPlayer` shows play/pause button (32×32 circle, `theme.accent` background) + `CardWaveformView`. Unlike recording waveform (scroll-based), card waveform is **stateless fill**: all 75 bars always visible with 2pt spacing (barSlot = 4pt), distributed across available width. Canvas redraws on `playbackProgress` change — bars to left of progress filled with `theme.accent`, rest with `#C3C3C3`. No TimelineView, no scroll — just progressive left-to-right color fill. Self-contained `CardAudioPlayer` (AVAudioPlayer wrapper) handles playback with AVAudioSession `.playback` mode + `fileTypeHint: .m4a`. `AudioFileAnalyzer.analyzeBars()` analyzes m4a files via `vDSP_rmsqv`, `cbrtf` curve, returns 75 bars (2-24pt height). If file unreadable (<100 bytes), falls back to `placeholderBars()` (random heights). **Performance:** `AudioAnalysisCache.shared` singleton caches analysis results by file path + deduplicates concurrent requests via `inFlight` tracking — prevents redundant file I/O when scrolling through audio dreams. On finish: resets `playbackProgress = 0`.
- **Press feedback:** `PressableButtonStyle` in `Theme/PressableButtonStyle.swift` — `scaleEffect(0.9)` + `opacity(0.7)` with 0.15s easeOut animation. Used on `DreamCardPlayer` play/pause and `TabBarItem`. For glass buttons, `.reveriGlass(.interactive())` provides native press feedback instead.
- **Toast system:** `ToastView` supports `ToastStyle` enum (`.success` = green icon, `.error` = red icon). View modifier: `.toast(isPresented:message:icon:style:duration:)`. Used in `DreamDetailView` for image generation error feedback.
- **Keyboard Done button:** RecordView adds `.toolbar { ToolbarItemGroup(placement: .keyboard) }` with Done button that dismisses both text mode and review mode focus states. Single toolbar on `voicePlaceholder` covers both TextEditors (they're in mutually exclusive `if/else` branches).
- **Liquid Glass style:** Unified modifier `.reveriGlass(.circle)` / `.reveriGlass(.capsule)` in `GlassButtonStyle.swift`. Uses native `.regular.tint(.white.opacity(0.15)).interactive()` — subtle milky tint + interactivity (like iOS Back button). Multiple glass elements nearby → wrap in `GlassEffectContainer(spacing:)`. Exception: overlapping circles (collapsed emotions) must NOT use `GlassEffectContainer` — each circle stays independent with `zIndex` for proper overlap.
- **Journal header:** Black background + blur gradient orb (orange day / blue night) at trailing edge. Top row: Profile button (SF Symbol `person.fill`, 44×44 glass circle with locale picker Menu) + search bar (magnifyingglass + "Search", glass capsule, 44pt min height) + calendar filter button (CalendarIcon 22×22, 44×44 glass circle with TimeRange menu), wrapped in `GlassEffectContainer(spacing: 12)`. Bottom row: Title "My Dreams" (`.fixedSize()`) + content (Dreams tab: `EmotionFilterBar`, Folders tab: "New Folder" glass button). VStack spacing 20pt. Bottom HStack has `.frame(height: 42)` for constant header height regardless of tab. New Folder button: FolderAddIcon 16×16 + text, 14pt horizontal padding, `.reveriGlass(.capsule)` with interactive glass. Triggers `showNewFolderAlert` binding from `JournalView`.
- **Emotion filter bar:** Single `ScrollView(.horizontal)` + `HStack` with animated spacing: `-collapsedOverlap` (16pt) when collapsed, `expandedSpacing` (6pt) when expanded. `ScrollViewReader` resets scroll to first item on collapse via `onChange(of: isExpanded)`. `.scrollDisabled(!isExpanded)` + `.scrollClipDisabled()` (prevents clipping last circle). `emotionOrder` state lives in `RootView` (passed as `@Binding` through JournalView → JournalHeader → EmotionFilterBar) to persist across tab switches — tracks MRU order, selected emotion moves to front. Each emotion: `.ultraThinMaterial` circle 42×42 + emotion image 30×30 + optional selected tint (`emotion.color.opacity(0.25)`) + dimming overlay (`.black.opacity(0.55)` for non-selected when filter active, `.id()` for instant appearance) + white gradient stroke (0.75pt). `zIndex` animated: collapsed uses `count - index` (first on top), expanded uses `0` (natural order). Tap: collapsed → expand, expanded → select/deselect + collapse. `.frame(maxWidth: .infinity, alignment: .leading)` + `.frame(height: circleSize)` ensures constant layout footprint — parent HStack never shifts during animation. Spring animation (0.4s, bounce 0.15) on `isExpanded` only.
- **Emotion filter → tab bar:** `selectedEmotionFilter` lives in `RootView`, passed as `@Binding` to `JournalView` (synced to `JournalViewModel.selectedEmotion` via `onChange`), and as value to `ReveriTabBar.emotionFilter`. When filter active AND `selectedTab == .journal`: journal tab shows `emotion.displayName` as label, `emotion.journalIcon` as icon, `emotion.color` as accent (text + background). Filter styling hidden when switching to other tabs (smooth spring animation), reappears when returning to Journal tab. Label stays expanded while filter is active on Journal tab.
- **Journal filtering performance:** `JournalViewModel` caches `filteredDreams` as a stored property instead of computed. Filter runs ONLY when filter parameters change (via `.onChange()` for `searchText`, `selectedEmotion`, `selectedTimeRange`, `allDreams`), not on every body render. Search text debounced 300ms to avoid filtering on every keystroke. Search uses `.localizedCaseInsensitiveContains()` (Foundation-optimized, no string allocations) instead of `.lowercased()`. With 100 dreams: ~95% reduction in filter operations vs computed property approach. `EmotionFilterBar` uses `emotionOrder.indices` instead of `Array(enumerated())` to avoid intermediate array allocation.
- **Emotion picker flow (post-save):** Save dream → `HowDidItFeelCard` appears immediately above tab bar. Two-phase card: first "How did it feel?" + X button, X dismisses → shows "Dream saved" (green #4CAF50, 10% bg / 100% text) for 1.5s → card slides away. Tapping card opens `EmotionPickerGrid` (inline floating grid, NOT a sheet). Tab bar switches to `savingFeelingsControls` — single "Save feelings" button (`CheckmarkBigIcon`), no journal icon. Selecting emotions → tap save → emotions saved to dream → "Dream saved" shown → dismiss. Tap outside grid → return to card. Auto-dismiss after 30s. `showHowDidItFeel`, `showEmotionGrid`, `showDreamSaved`, `pendingEmotions` states all in `RootView`. `RecordViewModel.onDreamSaved` closure wires dream reference. Card animations: appear = `spring + scale(0.8→1) + opacity`, disappear = `easeOut(0.35)`. No implicit `.animation()` on the card — all animation driven by call-site `withAnimation` in `RootView.dismissWithSaved()`. Transitions use `.opacity` (not `.blurReplace` — avoids expensive off-screen blur pass).
- **DetailDreamState:** `@Observable` class coordinating DreamDetailView ↔ ReveriTabBar communication. Lives in `RootView`, passed to both JournalView (forwards to DreamDetailView) and ReveriTabBar. Properties: `isActive` (view mounted), `hasInterpretation` (bool), `isGeneratingInterpretation` (bool), `interpretationError: String?`, `tabBarMode: DetailTabBarMode` enum (`.generateImage`, `.generateImageAgain`, `.interpretDream`, `.none`), `interpretTrigger: Bool` (toggle trigger, NOT callback). Pattern: DreamDetailView updates `tabBarMode` on tab switch / generation state change → ReveriTabBar reads mode to render correct button. TabBar toggles `interpretTrigger` → DreamDetailView catches via `onChange(of: detailState.interpretTrigger)` and calls service. Similar to existing `detailDreamGenerateTrigger` pattern for image generation. Avoids callback hell, leverages Observation framework.
- **EmotionPickerGrid:** 2-row grid (4+3): [confused, joyful, inLove, calm] / [anxious, scared, angry]. Each emotion: 44×44 glass circle (`.reveriGlass(.circle, interactive: false)`) with 32×32 SVG inside. Selection tint overlay + dimming (0.55 black) for unselected when any selected. Staggered appear animation: each item drops from above (`offset(y: -30)` + `scale(0.5→1)` + opacity) with 60ms delay per item, spring(0.5, bounce 0.35). `@Binding var selectedEmotions: Set<DreamEmotion>`.
- **Dream Reminder (Live Activity):** Automatic bedtime reminder via Live Activity on lock screen. **Flow:** User sets bedtime in ProfileView (e.g. 11 PM) → `NotificationService` schedules `UNCalendarNotificationTrigger` for each selected weekday → at bedtime, notification fires → if app in foreground: auto-starts Live Activity (suppresses banner), if app in background: shows notification "Time to Sleep", user taps → app opens → starts Live Activity → Live Activity persists overnight with Record/Write buttons on lock screen → user wakes up, taps Record/Write → deep link (`reveri://record` / `reveri://write`) opens app → after saving dream, `dreamReminderManager.end()` dismisses Live Activity. **Architecture:** `DreamReminderAttributes` (ActivityAttributes, shared with widget extension via file copy), `DreamReminderManager` (@Observable, lives in RootView — `start()`/`end()`/`reconnect()`/`refresh()`), `DreamReminderLiveActivity` (widget UI in `RecordingActivityWidget/`). **Notification → Live Activity bridge:** `NotificationDelegate.willPresent` (foreground) and `didReceive` (default action) post `.dreamReminderStartActivity` → RootView `.onReceive` calls `dreamReminderManager.start()`. Record/Write notification actions still post `.dreamReminderRecord`/`.dreamReminderWrite` for direct access. **Widget extension:** `RecordingActivityWidgetExtension` target, `PBXFileSystemSynchronizedRootGroup` in pbxproj, `Info.plist` with `NSExtension.NSExtensionPointIdentifier = com.apple.widgetkit-extension`, bundle ID `com.ReveriAI.ReveriAI.RecordingActivityWidget`.
- **Profile screen:** Push navigation from JournalHeader profile button. Custom nav bar (glass circle back button + "Profile" title). Two sections: (1) **Speech Recognition** — `SpeechLocale` picker via `Menu` (same `@AppStorage("speechRecognitionLocale")`), (2) **Dream Reminder** — Toggle + `DatePicker(.hourAndMinute)` + weekday selector (7 circles, Mon-Sun order, selected = accent fill). Settings: `@AppStorage` keys `reminderEnabled`, `reminderHour`, `reminderMinute`, `reminderDays` (comma-separated weekday ints). `onChange` handlers call `notificationService.scheduleReminders()`/`.cancelAllReminders()`. Permission requested on first toggle enable. Test notification button sends in 3s.
- **NotificationService:** `@Observable`, handles UNUserNotificationCenter permissions + scheduling + delegate setup. `setupDelegate()` called from `App.init()` (before any notification arrives). Separate `NotificationDelegate` (plain NSObject, not @Observable) for concurrency safety. Category `DREAM_REMINDER` with Record/Write actions (`.foreground` option). `Notification.Name` extensions: `.dreamReminderRecord`, `.dreamReminderWrite`, `.dreamReminderStartActivity`.
- **URL scheme deep links:** `reveri://` scheme registered in `Info.plist` (`CFBundleURLTypes`). Handled in `RootView.handleDeepLink(_:)` via `.onOpenURL`. Routes: `reveri://record` (start recording), `reveri://write` (text mode), `reveri://stop-recording` (stop active recording). Used by Live Activity `Link(destination:)` buttons and Recording Live Activity stop button.
- **Assets:** SVG icons stored with `preserves-vector-representation`. Tab icons use `template-rendering-intent: original` (colors in SVG). Header/mode icons use `template` rendering. `CheckmarkIconAction`, `CheckmarkBigIcon`, `StopIcon`, `PauseIcon`, `PlayIcon`, `DeleteIcon`, `SkipBack5Icon`, `SkipForward5Icon` use `original` rendering. Emotion SVGs (optimized from 4.1MB to 680KB via SVGO, original rendering, preserves-vector-representation): `EmotionJoyful`, `EmotionInLove`, `EmotionCalm`, `EmotionConfused`, `EmotionAnxious`, `EmotionScared`, `EmotionAngry`. Emotion tab SVGs (original rendering): `JournalIconJoyful`, `JournalIconInLove`, `JournalIconCalm`, `JournalIconConfused`, `JournalIconAnxious`, `JournalIconScared`, `JournalIconAngry`.
